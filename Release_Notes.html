Overview

  

•  The X-CUBE-AZURE (STM32 Azure IoT software expansion for STM32Cube) provides ready-to-run firmware examples to support quick evaluation and development of Azure IoT Cloud applications on B-L475E-IOT01, 32F413HDISCOVERY and 32F769IDISCOVERY boards.

•  Three IDE are supported: IAR, Keil and System Workbench.

•  It contains BSP compatible sensor drivers for B-L475E-IOT01.

•  It connects to Azure IoT hubs through an MQTT over TLS connection.

•  Below the most useful document:

º  UM2242 - Getting started with Microsoft® Azure IoT cloud software expansion for STM32Cube.

V1.0.1 / 5-October-2017

 

Main Changes

•  Updated dependencies (see contents list below).

•  Bug fixes.

º  Azure application: Fix the reconnection to the IoT hub after the expiration of the SAS token (after 48min by default).

–  Note: The Inventek ES_WIFI WiFi module on B-L475E-IOT01 and 32F413H-DISCO does not accept further TCP send messages once the remote host has closed the connection (FIN TCP tag). This causes a few lines of error log on the console when the SAS token expires (i.e. after 'Connection expired. Reconnecting' is displayed. But the Azure SDK is robust to this error and it reconnects correctly.

º  Azure application: Fix the crt initialization.

º  ES_WIFI driver (B-L475E-IOT01, 32F413H-DISCO): Solved random network disconnections that were due to a de-synchronization between the SPI and the GPIO.

º  Ethernet & TCP/IP configuration (32F769I-DISCO).

–  Generate a unique MAC address derived from the CPU ID in order to avoid address collisions when several F7 boards are connected to the same LAN.

–  Activate the local port number true randomization, to avoid possible TLS session confusions on server side after a board reset.

–  Activate the TCP keep-alive. Otherwise, the remote host may disconnect earlier than the first MQTT keep-alive message.

º  Improved error management.
